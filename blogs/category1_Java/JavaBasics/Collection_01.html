    # Collection接口和常用方法

    Public interface Collection<E> extends Interable <E><br>
    1

    1）collection 实现子类可以存放多个元素，每个元素可以是Object
    2）有些Collection的实现类，可以存放重复的元素，有些不可以
    3）有些Collection的实现类，有些是有序的（List），有些是无序的（Set）
    4）Collection接口没有直接的实现子类，是通过他的子接口Set和List来实现的
    Collection接口常用方法，以实现子类
    ArryList演示
    1)add:添加单个元素2) remove:删除指定元素3) contains:查找元素是否存在4) size:获取元素个数5) isEmpty:判断是否为空6) clear:清空7) addAll:添加多个元素8) containsAll:查找多个元素是否都存在9) removeAll:删除多个元素10)说明:以ArrayList实现类来演示

    public class CollectionMethod {
    @SuppressWarnings({"all"})
    public static void main(String[] args) {
        List list = new ArrayList();
//        1)add:添加单个元素
        list.add("jack");
        list.add("tom");
        list.add(999);
        list.add(10);
        list.add(true);
        list.add(false);
        System.out.println(list);//[jack, tom, 999, 10, true, false]

//        2) remove:删除指定元素
        list.remove(0);//删除第一个元素
        list.remove(true);
        System.out.println(list);//[tom, 999, 10, false]

//        3) contains:查找元素是否存在
        System.out.println(list.contains(999));//true
        System.out.println(list.contains("%"));//false

//        4) size:获取元素个数
        System.out.println(list.size());//4

//        5) isEmpty:判断是否为空
        System.out.println(list.isEmpty());//false

//        6) clear:清空
//        list.clear();
//        System.out.println(list);

//        7) addAll:添加多个元素
        ArrayList list2 = new ArrayList();
        list2.add("1");
        list2.add("2");
        list.addAll(list2);
        System.out.println(list);//[tom, 999, 10, false, 1, 2]

//        8) containsAll:查找多个元素是否都存在
        System.out.println(list.containsAll(list2));//true

//        9) removeAll:删除多个元素
        list.add("5");
        list.removeAll(list2);
        System.out.println(list);//[tom, 999, 10, false, 5]
    }
}

    # Collection接口遍历元素的方式1-使用Iterator（迭代器）

    1. Iterator对象称为迭代器，主要用于遍历Collection集合中的元素。
    2. 所有实现了Collection接口的集合类都有一个iterator()方法，用以返回一个实现了lterator接口的对象，即可以返回一个迭代器。
    3. Iterator的结构。
    4. Iterator仅用于遍历集合,Iterator本身并不存放对象。

    # 迭代器执行原理

    Iterator  iterator  =  coll.iterator（）；//得到一个集合的迭代器
    //hasNext():判断是否还有下一个元素
    While（iterator.hasNext（））{
    //next（）：指针下移；将下移以后集合位置上的元素返回
        System.out.println(iterator.next());
    }
    Iterator接口方法 hasNext（），next（），remove（）
    在调用iterator.next（）方法前必须调用iterator.has.next（）进行检测。若不调用，且下一条记录无效，直接调用it.next（）会抛出NoSuchElementException异常。

    # Collection接口遍历元素的方式2-for循环增强

    增强for循环，可以代替iterator迭代器， 特点:增强for就是简化版的iterator , 本质一样。只能用于遍历集合或数组。

    # 基本语法

    for(元素类型元素名:集合名或数组名) {
        访问元素
    }

    Collection collection = new ArrayList();
collection.add(new Book("一","1",10.1));
collection.add(new Book("二","2",5.1));
collection.add(new Book("三","3",4.1));

//1.使用增强for循环
//2.增强for的底层任然是迭代器
//3.增强for可以理解为简化版的 迭代器遍历
//快捷方式  I
for (Object book: collection) {
    System.out.println(book);
}
//增强for也可直接在数组使用
int[] nums = {1,8,10,90};
for (int i:nums) {
    System.out.println(i);
}
