    # 异常处理

    异常处理就是当异常发生时，对异常的处理方式

    # 处理方式

    # 1. try-catch-finally

    程序员在代码中捕获发生的异常，自行处理。可以根据需要在程序中有多个try catch
    try-catch-finally处理机制：
    try{
    代码有可能异常
    }catch{
    //捕获到异常
    //1.当异常发生时
    //2.系统将异常封装成Exception 对象 e，传递给catch
    //3.得到异常对象后，程序员自己处理
    //4.如果没有发生异常，catch代码块不执行
    }finally{
    //1.不管try代码块是否有异常发生，始终要执行finally
    //2.所以，通常将释放资源的代码，放在finally
    }
    如果没有finally也可

    # 细节

    1. 如果异常发生了，则异常发生后面的代码不会执行，直接进入到catch块
    2. 如果异常没有发生，则顺序执行try的代码块，不会进入到catch.
    3. 如果希望不管是否发生异常，都执行某段代码(比如关闭连接，释放资源等)则使用如下代码finally {}
    4. 可以有多个catch语句，捕获不同的异常(进行不同的业务处理)，要求父类异常在后，子类异常在前，比如(Exception 在后，NullPointerException 在前)，如果发生异常，只会匹配一个catch
    5. 可以进行try -finally 配合使用，这种方法相当于没有捕获异常，因此程序会直接崩溃/退出 try{
      }finally{
      }
    public class TryCatchDetail {
    public static void main(String[] args) {
        try {
            String str = "一二三";
            int a = Integer.parseInt(str);
            System.out.println(a);
        } catch (NumberFormatException e) {
            System.out.println("异常信息"+e.getMessage());
        }finally{
            System.out.println("finally执行");
        }
    }
}

public class TryCatchDetail02 {
    public static void main(String[] args) {
        try {
            Person person = new Person();
            person = null;
            System.out.println(person.getName());
            int n1 = 10;
            int n2 = 0;
            int res = n1 / n2;
        } catch (ArithmeticException e) {
            System.out.println("算数异常" + e.getMessage());
        } catch (NullPointerException e) {
            System.out.println("空指针异常" + e.getMessage());
        } catch (Exception e) {
            System.out.println(e.getMessage());
        } finally {
            System.out.println("123");
        }
    }
}

    private String name = "jack";

    public String getName() {
        return name;
    }

}

    # 2. throws

    将发生的异常抛出，交给调用者（方法）来处理,最顶级的处理者是JVM （若未显示使用trycatch，默认使用throws）
    throws处理机制：

    如果一个方法（中的语句执行时）可能产生异常，但是并不能确定如何处理这种异常，则此方法应显示的声明抛出异常，表明该方法将不对这些异常进行处理，而由该方法的调用者负责处理。
    在方法声明中用throws语句可以声明抛出异常的列表，throws后面的异常类型可以是方法中产生的异常类型，也可以是它的父类

    # 细节

    1. 对于编译异常，程序中必须处理，比如try-catch或者throws
    2. 对于运行时异常，程序中如果没有处理，默认就是throws的方式处理
    3. 子类重写父类的方法时，对抛出异常的规定:子类重写的方法，所抛出的异常类型要么和父类抛出的异常一致，要么为父类抛出的异常的类型的子类型
    4. 在throws过程中，如果有方法try-catch ,就相当于处理异常，就可以不必throws。
    public static void f1() throws FileNotFoundException{
   //因为f3（）方法抛出一个编译异常
   //即这时，就要求f1（）必须处理这个编译异常
   f3();//抛出异常  try catch finally 或者 继续throws （public static void f1() throws RuntimeException）
}
public static void f3() throws FileNotFoundException {
   FileInputStream fis = new FileInputStream("d:aa.txt");
}
public static void f4(){
   //f4()可调用f5（）
   //因为f5是运行异常
   //java中，并不要求程序员显示处理，因为有默认处理
   f5();
}
public static void f5() throws ArithmeticException{
}

    # 自定义异常

    当程序中出现了某些“错误”，但该错误信息并没有在throwsable子类中描述处理，这个时候可以自己设计异常类，用于描述该错误信息

    1. 定义类:自定义异常类名(程序员自己写)继承Exception或RuntimeException
    2. 如果继承Exception,属于编译异常
    3. 如果继承RuntimeException,属于运行异常。（一般来说，继承RuntimeException）
    public class CustomException {
    public static void main(String[] args) {
        int age = 999;
        //要求在18-120之间，否则抛出异常
        if (!(age >=18&&age<=120)){
            //通过构造器，设置信息
            throw new AgeException("需要在18-120之间");
        }
        System.out.println("范围正确");
    }
}
//自定义异常
//一般情况下我们自定义异常是继承RuntimeException
//即把自定义异常作为运行时异常
//好处是可以用默认处理机制，方便
class AgeException extends RuntimeException{
    public AgeException(String message) {//构造器
        super(message);
    }
}
