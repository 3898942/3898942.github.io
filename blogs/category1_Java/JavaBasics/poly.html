    Java三大特性——多态

    lpy 12/12/2021 多态

    # 多态

    目的：提高代码复用性。
    建立在封装和继承基础之上

    1. 方法的多态   重写和重载就体现多态

    2. 对象的多态（核心）     一个对象的编译类型和运行类型可以不一致
          Animal animal = new Dog();
          animal编译类型是Animal，运行类型是Dog，Animal是Dog的父类，父类的对象引用指向子类的对象
          编译类型在定义对象时，就确定了，不能改变
          运行类型是可以变化的

          animal = new Cat();
          animal的运行类型变成的Cat，编译类型仍然是Animal
          编译类型看定义时 = 号的左边，运行类型看 = 的右边

    # 多态使用细节

    多态的前提：继承关系

    # 多态的向上转型

    本质：父类的引用指向了子类的对象
    语法:父类类型 引用名 = new 子类类型（）；
    特点:

    1. 编译类型看左边，运行类型看右边
    2. 可以调用父类中所有的成员（需遵守访问权限）
    3. 不能调用子类中特有的成员
    4. 最终运行结果看子类的具体实现

    # 多态的向下转型

    语法 子类类型  引用名  =  （子类类型）  父类引用；
    特点:

    1. 只能强转父类的引用，不能强转父类的对象
    2. 要求父类的引用必须指向的是当前目标类型的对象
    3. 向下转型后，可以调用子类类型中所有的成员

    instanceof 用于判断对象的运行类型是否为某个类的类型或是其子类型

    	//已知class BB extends AA
	AA a =new BB();
	//a运行类型是BB
	System.out.println(a instanceof BB);//a运行类型是BB,是BB类
	System.out.println(a instanceof AA);//a运行类型是BB,是AA类的子类
	System.out.println(a instanceof Object);//a运行类型是BB,是Object类或是其子类

    public class PolyEetail {
    public static void main(String[] args) {

        //向上转型：父类的引用指向了子类的对象
        Object obj = new Cat();
        Animal animal = new Cat();
        //可以调用父类中所有的成员（需遵守访问权限）
        //不能调用子类中特有的成员
        //编译阶段，能调用哪些成员，由编译类型决定
        //animal.catchMouse()；错误
        //最终运行结果看子类（运行类型）的具体实现，即调用方法时，按照从子类开始查找方法
        //规则与方法调用规则一致
        animal.eat();//输出猫吃鱼
        animal.run();
        animal.show();
        animal.sleep();

        //向下转型
        //1.语法：子类类型  引用名 = （子类类型） 父类引用；
        //cat的编译类型是Cat，运行类型是Cat
        Cat cat = (Cat) animal;
        cat.catchMouse();
        //2.父类的引用必须指向的是当前目标类型的对象
        //Dog dog = （Dog）animal;//错误

        System.out.println("ok");

        Animal animal1 = new  Dog();
        Dog dog = (Dog) animal1;
        dog.say();
    }
}
public class Animal {
   String name = "动物";
   int age = 10;
   public void sleep(){
      System.out.println("sleep");
   }
   public void run(){
      System.out.println("run");
   }
   public void eat(){
      System.out.println("eat");
   }
   public void show(){
      System.out.println("shwo");
   }

}
public class Cat extends Animal{
   public void eat(){//方法重写
      System.out.println("猫吃鱼");
   }
   public void catchMouse(){
      System.out.println("猫抓鼠");
   }
}
public class Dog extends Animal{
   public void say(){
      System.out.println("wang");
   }
}

    # 动态绑定机制

    1. 当调用对象方法的时候，该方法会和对象的内存地址/运行类型绑定
    2. 当调用属性时，没用动态绑定机制，哪里声明，哪里使用
    public class DynamicBinding {
    public static void main(String[] args) {
        //a的编译类型为A，运行类型为B
        A a = new B();//向上转型
        System.out.println(a.sum());//30
        System.out.println(a.sum1());//20
    }
}
class A {
    int i = 10;
//  java动态绑定机制
//  1.当调用对象方法的时候，该方法会和对象的内存地址/运行类型绑定
//  2当调用属性时，没用动态绑定机制，哪里声明，哪里使用
    public int sum() {
        return getI() + 10;//动态绑定机制，调用子类getI，20+10
    }

    public int sum1() {
        return i + 10;
    }

    public int getI() {
        return i;
    }
}
