    # 创建线程的多种方式

    # Lock接口

    lock实现了提供比synchronized方法和语句更广泛的锁定操作

    lock是一个类，synchronized是一个接口。

    synchronized不需要用户手动释放锁，系统自动释放；lock需用户手动释放锁。

    lock实现可重入锁：

    public class ticksell {
    public static void main(String[] args) {
        Tick tick = new Tick();
        new Thread(()->{
            for (int i = 0; i < 40; i++) {
                tick.sale();
            }
        },"AA").start();
        new Thread(()->{
            for (int i = 0; i < 40; i++) {
                tick.sale();
            }
        },"BB").start();
        new Thread(()->{
            for (int i = 0; i < 40; i++) {
                tick.sale();
            }
        },"CC").start();
    }
}
class Tick{
    private int number= 30;
    //创建可重入锁
    private final ReentrantLock lock = new ReentrantLock();
    //售票
    public void  sale(){
        //上锁
        lock.lock();
        try {
            if (number >0){
                number--;
                System.out.println(Thread.currentThread().getName()+":卖出"+"剩余："+number);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }finally {
            //解锁
            lock.unlock();
        }
    }
}

    # 线程间通信 （synchronized）与虚假唤醒问题

    //第一步创建资源类
class Share{
    //初始值
    private int number = 0;
    public synchronized void incr() throws InterruptedException {
        //第二步
        //number为0，加1，不为0，等待
        if (number != 0){
            wait();//wait()方法并不是在等待资源的锁，而是在等待被唤醒。
        }
        number++;
        System.out.println(Thread.currentThread().getName()+"::"+number);
        //通知其他线程
        this.notifyAll();
    }
    public synchronized void decr() throws InterruptedException {
        if (number != 1){//number
            wait();
        }
        number--;
        System.out.println(Thread.currentThread().getName()+"::"+number);
        this.notifyAll();
    }
}

    1. 当A抢到锁时，num=0，往下执行num = 1，并且唤醒其他线程。
    2. 这时C线程抢到锁，发现num=1，不能再加1了，this.wait等待后释放锁。
    3. 这时线程A又抢到锁了，发现num=1，还是不能进行+1操作，this.wait等待后释放锁。
    4. B线程抢到锁后，num=1，可以进行-1操作，并notifyAll唤醒其他线程。
    5. A被唤醒抢到锁，wait有个特点，从哪里等待就从哪里继续执行。这时的num=0，进行+1操作，并唤醒其他线程。
    6. C被唤醒抢到锁，继续往下执行，num+1，所以结果num=2，

    因此要将if改为while，避免虚假唤醒。

    class Share{
    //初始值
    private int number = 0;
    public synchronized void incr() throws InterruptedException {
        //第二步
        //number为0，加1，不为0，等待
        while (number != 0){
            wait();//wait()方法并不是在等待资源的锁，而是在等待被唤醒。
        }
        number++;
        System.out.println(Thread.currentThread().getName()+"::"+number);
        //通知其他线程
        this.notifyAll();
    }
    public synchronized void decr() throws InterruptedException {
        while (number != 1){//number
            wait();
        }
        number--;
        System.out.println(Thread.currentThread().getName()+"::"+number);
        this.notifyAll();
    }
}

    # 线程间通信（lock）

    class Share1{
    private int number = 0;

    private Lock lock = new ReentrantLock();
    private Condition condition = lock.newCondition();

    public  void incr() throws InterruptedException {
        //上锁
        lock.lock();
        try {
            while (number != 0){
                condition.await();
            }
            number++;
            System.out.println(Thread.currentThread().getName()+"::"+number);
            condition.signalAll();

        } finally {
            //解锁
            lock.unlock();
        }
    }
    public  void decr() throws InterruptedException {
        //上锁
        lock.lock();
        try {
            while (number != 1){
                condition.await();
            }
            number--;
            System.out.println(Thread.currentThread().getName()+"::"+number);
            condition.signalAll();
        } finally {
            //解锁
            lock.unlock();
        }
    }
}

    class Share3 {
    private int flag = 1;//标志位
    private Lock lock = new ReentrantLock();
    private Condition condition1 = lock.newCondition();
    private Condition condition2 = lock.newCondition();
    private Condition condition3 = lock.newCondition();

    public void print5(int loop) throws InterruptedException {
        lock.lock();
        try {
            while (flag != 1) {
                condition1.await();
            }
            for (int i = 0; i < 5; i++) {
                System.out.println(Thread.currentThread().getName() + "::" + i + ":轮数" + loop);
            }
            //通知
            flag = 2;//修改标志位为2
            condition2.signal();
        } finally {
            lock.unlock();
        }
    }

    public void print10(int loop) throws InterruptedException {
        lock.lock();
        try {
            while (flag != 2) {
                condition2.await();
            }
            for (int i = 0; i < 10; i++) {
                System.out.println(Thread.currentThread().getName() + "::" + i + ":轮数" + loop);
            }
            //通知
            flag = 3;//修改标志位
            condition3.signal();
        } finally {
            lock.unlock();
        }
    }

    public void print15(int loop) throws InterruptedException {
        lock.lock();
        try {
            while (flag != 3) {
                condition3.await();
            }
            for (int i = 0; i < 15; i++) {
                System.out.println(Thread.currentThread().getName() + "::" + i + ":轮数" + loop);
            }
            //通知
            flag = 1;//修改标志位为2
            condition1.signal();
        } finally {
            lock.unlock();
        }
    }
}

    # synchronized锁

    普通同步方法，对单个对象加锁

    作用于静态方法时，其锁就是当前类的class对象锁

    使用同步代码块，可以使用同步代码块的方式对需要同步的代码进行包裹，这样就无需对整个方法进行同步操作

    一个对象在两个线程中分别调用一个静态同步方法和一个非静态同步方法，不会产生互斥。因为虽然是一个对象调用，但是两个方法的锁类型不同，调用的静态方法实际上是类对象在调用，即这两个方法产生的并不是同一个对象锁，因此不会互斥，会并发执行。

    # 公平锁、非公平锁

    公平锁：按照申请锁的顺序来获取锁，不会出现争抢的情况。

    非公平锁：多线程不会按照顺序获取锁，会先尝试去争抢锁，有可能后申请的线程比先申请的线程先抢到锁，在高并发情况下，可能会造成优先级反转和饥饿的现象。(例如Re entran tLock默认为非公平锁，可设置参数为true变为公平锁ReentrantLock lock = new ReentrantLock(true))

    区别：公平锁就是很公平，争抢锁的几率一样，每个线程会先看等待队列是否为空，若为空，直接获取锁，若不为空，自动排队等候获取锁；非公平锁就是所有的线程都会优先去尝试争抢锁，不会按顺序等待，若抢不到锁，再用类似公平锁的方式获取锁。

    # 可重入锁（递归锁）

    synchronized（隐式）和Lock（显示）都是可重入锁，即一个线程可以重复获取同一把锁

    # 死锁

    两个或两个以上的线程在执行过程中，因为争夺资源而造成一种互相等待的现象。

    • 产生死锁的四个必要条件

      • 互斥条件；多个线程不能使用统一资源。如果线程 A 已经持有的资源，不能再同时被线程 B 持有，如果线程 B 请求获取线程 A 已经占用的资源，那线程 B 只能等待，直到线程 A 释放了资源。

      • 持有并等待条件；当线程 A 已经持有了资源 1，又想申请资源 2，而资源 2 已经被线程 C 持有了，所以线程 A 就会处于等待状态，但是线程 A 在等待资源 2 的同时并不会释放自己已经持有的资源 1。

      • 不可剥夺条件；当线程已经持有了资源 ，在自己使用完之前不能被其他线程获取，线程 B 如果也想使用此资源，则只能在线程 A 使用完并释放后才能获取。

      • 环路等待条件；线程 A 已经持有资源 2，而想请求资源 1， 线程 B 已经获取了资源 1，而想请求资源 2，这就形成资源请求等待的环形图。

    • 避免死锁

      • 资源有序分配

      线程 A 和 线程 B 获取资源的顺序要一样，当线程 A 是先尝试获取资源 A，然后尝试获取资源 B 的时候，线程 B 同样也是先尝试获取资源 A，然后尝试获取资源 B。也就是说，线程 A 和 线程 B 总是以相同的顺序申请自己想要的资源。

    public class DeadLock {
    static Object a = new Object();
    static Object b = new Object();

    public static void main(String[] args) {
        new Thread(() -> {
            synchronized (a) {
                System.out.println(Thread.currentThread().getName() + "持有锁a，试图获取锁b");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (b) {
                    System.out.println(Thread.currentThread().getName() + "获取锁b");
                }
            }
        }, "A").start();

        new Thread(() -> {
            synchronized (b) {
                System.out.println(Thread.currentThread().getName() + "持有锁b，试图获取锁a");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (a) {
                    System.out.println(Thread.currentThread().getName() + "获取锁a");
                }
            }
        },"B").start();
    }
}
